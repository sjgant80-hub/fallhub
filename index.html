How it works

Four steps. One day. Config beats code.

OneHub is one static-HTML frontend + one Cloudflare Worker + one config file. Everything an operator would customise lives in the config — not the code.

STEP 1

Fork the repo

Click Fork on GitHub. You now have your own copy at github.com/you/onehub. MIT licence · yours forever.

STEP 2

Edit one config

Open config/onehub.config.json. Change business name, units, prices, brand colours, channel iCal URLs. Save.

STEP 3

Deploy the Worker

Run npm run deploy. Cloudflare Workers publishes your webhook endpoint. Free tier easily covers a small operator.

STEP 4

Enable Pages

Switch GitHub Pages on for the repo. Book page + Hub page go live at your-brand.github.io/onehub/. Point your domain at it.

Deploy in ~30 minutes

Cloudflare Worker + GitHub Pages. Free tier.

Two commands, one config, one DNS record.

1 · The Worker (webhooks)

Handles Stripe payments, WhatsApp inbound, iCal fetching, email inbound.

git clone https://github.com/YOU/onehub
cd onehub
npm install
cp worker/wrangler.example.toml worker/wrangler.toml
# edit worker/wrangler.toml · paste your keys
npm run deploy:worker

Worker publishes at your-onehub.workers.dev. Point api.yourdomain.com at it in Cloudflare DNS.

2 · The frontend (Pages)

Book page, hub page, admin page — static HTML, no build step.

cp config/onehub.config.example.json config/onehub.config.json
# edit config/onehub.config.json for your business
git commit -am "my config"
git push
# Enable GitHub Pages: Settings → Pages → main / (root)

Pages publishes at yourname.github.io/onehub/. Point yourdomain.com at it.

FAQ

The questions everyone asks.

Do I need to know how to code?
To fork and edit the config file — no. Every operator-specific value lives in config/onehub.config.json which is a text file you edit like a spreadsheet. To modify HTML or add custom features — you'd need basic HTML/JS, or hire someone (Simon takes bespoke work · see FallEnterprise tier ladder).
How is this different from Hostaway / Lodgify / Guesty?
Those charge £79-£200/month forever. You never own the software. If they go bankrupt or price-hike, you're stuck. OneHub is a one-time fork · MIT-licensed · yours forever · zero monthly fees. Stripe takes 1.4% on direct bookings, that's it.
What about Airbnb's 15% commission?
Bookings that come IN via Airbnb still pay Airbnb their commission — that's Airbnb's business, not ours. OneHub's win is DIRECT bookings via your own site (1.4% Stripe fee vs 15% Airbnb) plus unified inbox management for the bookings that do come via OTAs.
What happens if you disappear tomorrow?
MIT-licensed. You have the source. Any competent web dev can operate the codebase from the docs alone. Cloudflare Workers keeps running. GitHub Pages keeps serving. Nothing depends on OneHub-the-company continuing to exist.
Can I hire someone to set it up for me?
Yes. Two options: (1) hire Simon direct via Fiverr or email — typically £599-£3,499 for a full white-labelled setup. (2) fork the repo, hand it to any freelance developer with the credentials.html checklist filled — most competent devs can wire it up in 1-2 days.
Does the AI reply drafting need a paid API key?
No. Default is browser-native WebLLM — a small language model that runs entirely in your browser after a one-time ~600MB download. Free, works offline, slower on the first response. If you want faster/higher-quality replies, plug in a Claude or OpenAI API key and OneHub uses that instead.
Where does guest data live?
In the browser's IndexedDB on the device you're using as your operator hub (typically your phone or laptop). Plus a backup copy in your Cloudflare Worker's KV store if you enable that. Never on someone else's server unless you deliberately configure it that way. Sovereign by default.
What's the credentials.html file for?
A rendered setup checklist page you ship to a client operator (or use yourself). Walks through gathering every API key, iCal URL, and account credential in a structured form. See Wishwood's credentials.html for the live pattern.
